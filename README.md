# DSA to GitHub Extension

A Chrome extension that automatically pushes your Data Structures and Algorithms solutions from multiple coding platforms to your GitHub repository.

## 🚀 Features

### Multi-Platform Support
- **LeetCode** - Automatic detection of successful submissions
- **GeeksforGeeks** - Monitors "Problem Solved Successfully" messages  
- **TakeUforward** - Integrates with submission system

### Smart Organization
- Organized folder structure by platform and difficulty
- Automatic README generation for each problem
- Performance metrics tracking
- Solution file formatting with comments

### GitHub Integration
- Secure Personal Access Token authentication
- Automatic commit messages with problem details
- Support for custom branches
- Repository validation and testing

### User Experience
- Real-time submission detection
- Manual push buttons for each platform
- Cross-platform statistics tracking
- Clean, modern popup interface

## 📦 Installation

### From Source
1. Clone this repository
2. Open Chrome and go to `chrome://extensions/`
3. Enable "Developer mode"
4. Click "Load unpacked" and select the extension folder
5. The extension icon will appear in your toolbar

### Setup
1. Click the extension icon to open the popup
2. Go to the Configuration tab
3. Generate a GitHub Personal Access Token:
   - Visit [GitHub Token Settings](https://github.com/settings/tokens/new)
   - Select `repo` permissions
   - Copy the generated token
4. Fill in your GitHub credentials:
   - Personal Access Token
   - GitHub Username
   - Repository Name
   - Branch (default: main)
5. Click "Test Connection" to verify
6. Click "Save Configuration"

## 🎯 Usage

### Automatic Mode
1. Configure the extension with your GitHub credentials
2. Solve problems on any supported platform
3. Submit your solution
4. The extension automatically detects successful submissions
5. Your solution is pushed to GitHub with organized structure

### Manual Mode
- Look for the "📤 Push to GitHub" button on problem pages
- Click to manually push your current solution
- Useful for pushing existing solutions or when auto-detection fails

## ⚙️ Configuration

### Debug Mode
Control console logging for troubleshooting:
- **Enabled** - Shows detailed debug logs in browser console
- **Disabled** - Clean console output (default)

Access via extension popup → Configuration → "Enable Debug Logs" checkbox

### GitHub Token Permissions
Your Personal Access Token needs:
- `repo` - Full repository access
- `workflow` (optional) - For GitHub Actions integration

## 🐛 Troubleshooting

### Token Issues
- Ensure token has `repo` permissions
- Check if token is expired
- Verify token format (starts with `ghp_`)

### Repository Access
- Confirm repository exists
- Check username/repo name spelling
- Ensure you have push access

### Platform Detection
- Refresh the page after installing
- Check if you're logged into the platform
- Try manual push button if auto-detection fails

### Code Extraction
- Ensure you've written code before submitting
- Try manual push for existing solutions
- Check browser console for errors
